    Внесување модул

    Со тоа што програмата ви станува подолга, веројатно ќе сакате да ја поделите на повеќе датотеки за полесно одржување. Исто така, можеби ќе сакате да користите корисна функција што сте ја напишале во неколку програми, без да ја копирате дефиницијата на секоја од нив. Во Python, модулите се едноставно Python датотеки со .py екстензија кои содржат Python дефиниции и наредби. Модулите се внесуваат од други модули користејќи го зборот import и името на датотеката без .py екстензија. Пример, претпоставете дека напишавте скрипта наречена my_funcs.py со неколку функции (func1, func2 итн.). Сега, ако сакате да ги користите тие функции во друга скрипта која е сместена во истата директорија, можете да направите import my_funcs. Ова не ги внесува имињата на функциите дефинирани во my_funcs директно, но користејќи го името на модулот, можете да пристапите до функциите, на пример: 

    
      my_funcs.func1()

    Модулите можат да внесуваат други модули. Обично, но не е задолжително, сите наредби за внесување се ставаат на почетокот на модулот.
    Во кодот во едиторот, внесете го модулот calculator и креирајте инстанца од класата Calculator (calc). Користете ја методата add дефинирана во Calculator во јамка за да соберете броеви од 0 до 99.

         """


def hello_world(name):
      print(f"Hello, World! My name is {name}")

    Вградени модули

    Пајтон доаѓа со библиотека на стандардни модули. Некои модули се вградени во интерпретерот; тие обезбедуваат пристап до операции кои не се дел од срцето на јазикот, но се вградени за ефикасност или за пристап до примитиви на оперативниот систем, како системски повици. Еден посебен модул заслужува посебно внимание: sys, кој е вграден во секој интерпретер на Пајтон. Променливите sys.ps1 и sys.ps2 ги дефинираат низите кои се користат како примарни и секундарни промпти, ако интерпретерот е во интерактивен режим: 

    
      >>> import sys
>>> sys.ps1
'>>> '
>>> sys.ps2
'... '

    Променливата sys.path е список од низи кои одредуваат патеката за пребарување на модули на интерпретерот: видете што печати кога ќе го извршите кодот на задачата. Запомнете дека можете да користите Ctrl+Space по точка (.) за истражување на достапните методи на модул. 

    
      #Испечатете ја тековната дата користејќи го внесениот вграден модул datetime.
      import sys
      import datetime

      print(sys.path)

    Oд внесување

    Една форма на import наредбата ги внесува имињата од модулот директно. На овој начин, можете да го користите внесеното име без префиксот module_name. На пример: 

    
      from calculator  import Add

      calc = Add()     # Името Add се користи директно без префиксот calculator`

    Ова не го внесува името на модулот од кој се земаат увозите во локалната табела на симболи (па така, во нашиот пример, calculator не е дефиниран). Има и опција да се увезат сите имиња што ги дефинира модулот: 
    
      from calculator import *
      calc = Multiply()

    Ова ги увезува сите имиња, освен оние кои започнуваат со долница _ . Во повеќето случаи, Python програмерите не го користат ова, бидејќи внесува непознат сет на имиња во интерпретерот, што може да ги скрие некои од нештата што веќе сте ги дефинирале. Ако зад името на модулот следи as, тогаш името што следи по as се поврзува директно со увезаниот модул: 
    
      import my_module as mm
      mm.hello_world()

    Ова на практика увезува модулот на истиот начин како што import my_module ќе направи, со единствената разлика што е достапен како mm. Ова може да се користи и кога се користи from со слични ефекти: 
    
      from calculator import Subtract as Minus

    Пакетите

    Пакетите се начин на структурирање на просторот на имиња на модулите во Python користејќи "точкасто име на модул". На пример, името на модул A.B го означува подмодулот B во пакетот наречен A. Исто како што користењето на модулите ги заштитува авторите на различни модули од загриженост околу имињата на глобалните променливи, користењето на точкасти имиња на модули ги заштитува авторите на повеќемодулни пакети како NumPy или Pillow од загриженост околу имињата на модулите еден на друг. Проверете ги функциите и класите на пакетот кои ги создадовме. Корисниците на пакетот можат да увезуваат поединечни модули од пакетот, на пример: 

    
      import functions.greeting.hello

    Ова го вчитува подмодулот functions.greeting.hello. Тој мора да биде споменат со неговото целосно име: 
    
      functions.greeting.hello.hello('Susan')

    Алтернативен начин за увоз на подмодулот е: 
    
      from functions.greeting import hello

    Ова исто така увози подмодулот hello и го прави достапен без префиксот на пакетот, па може да се користи на следниот начин: 
    
      hello.hello('Susan')

    Во кодот, правилно увезете го официјалниот модул за да овозможите правилно функционирање на последната наредба за печатење. Во втората наредба за печатење, додадете повик на функција (пронајдете ја соодветната функција) за да испечатите збогум на 'Alex'.

    Извршување на модули како скрипти

    Еден Python модул е датотека која содржи извршни наредби како и дефиниции на функции или класи. Овие наредби се извршуваат кога името на модулот се сретнува во наредба за внесување (import). Името на датотеката е името на модулот со додаден суфикс .py. Внатре во модулот, името на модулот (како стринг) е достапно како вредност на глобалната променлива name. Кога ќе извршите модул директно (тоа е, не преку внесување во друг модул), кодот во модулот ќе се изврши, како да го внесувате, но со name поставено на "main". Можете да користите name и main на следниот начин: 

    
      if __name__ == "__main__":
       # Направи нешто овде

    Наредбите внатре во овој блок ќе бидат извршени само ако модулот се извршува директно, а не преку внесување во друг модул. На пример, да ги земеме две датотеки: main_program: 
    
      import some_module

      print(f"main_program __name__ is: {__name__}")

      if __name__ == "__main__":
      print("main_program executed directly")
      else:
      print("main_program executed when imported")

    some_module: 
    
      print(f"some_module __name__ is: {__name__}")

      if __name__ == "__main__":
      print("some_module executed directly")
      else:
      print("some_module executed when imported")

    Излезот по извршувањето на главниот програмски код: 
    
      some_module __name__ is: some_module
      some_module executed when imported
      main_program __name__ is: __main__
      main_program executed directly

    Излезот по извршувањето на some_module: 
    
      some_module __name__ is: __main__
      some_module executed directly

    Датотеките во оваа задача имаат исто име како во примерите погоре, но нивниот код е малку различен. Променете ги task.py и some_module.py така што кога ќе го извршите task.py, вашето излез ќе биде како следниот: 
    
      This is a message from some_module.
      This is a message from __main__.
      This is a message from the function in the imported module.
      This should be printed ONLY when task.py is called directly.
